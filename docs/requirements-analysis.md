# 万能视频下载站需求分析

## 1. 背景

很多用户希望把公开视频保存到本地，用于学习、资料归档、素材整理或离线观看。但平台原生下载能力常见问题包括：不支持下载、不支持批量、清晰度受限、移动端操作不方便、需要多个平台来回切换。

本项目目标是做一个跨平台、轻量、手机友好的万能视频下载网站。第一版先跑通核心链路，再逐步叠加付费能力。

## 2. 用户目标

- 粘贴视频链接后快速解析视频信息。
- 选择常用清晰度并下载到本地。
- 能在手机浏览器上完成下载流程。
- 对高级能力有明确预期，例如 4K、批量、字幕、翻译、AI 总结。
- 对长视频学习场景，希望快速获得视频大纲、核心知识点、时间戳字幕，并能继续向 AI 追问。

## 3. MVP 范围

第一版只做核心业务流程：

- 单链接解析。
- 展示标题、封面、作者、时长、平台和清晰度选项。
- 创建异步下载任务。
- 前端轮询进度，展示百分比、速度和剩余时间。
- 下载完成后保存文件到本地。
- 临时文件过期自动清理。
- 页面内预留 Pro 付费引导，但暂不接真实支付。

第一版下载 MVP 不实现：

- 账号系统。
- 数据库。
- 真实支付。
- 批量下载。
- cookies 上传。
- 字幕翻译。
- 历史记录。

当前 AI 总结迭代已补充实现：

- 对带平台原生字幕的视频创建 AI 总结任务。
- 生成视频总览、结构化要点、带时间戳字幕和思维导图。
- 支持围绕视频内容继续追问。
- 每 IP 每天 5 次免费额度，单视频最长 40 分钟。

## 4. 关键用户流程

1. 用户打开首页。
2. 用户粘贴公开视频链接。
3. 用户点击“立即解析”。
4. 后端用 `yt-dlp` 获取视频信息。
5. 前端展示视频信息和清晰度选项。
6. 用户选择清晰度并点击“开始下载”。
7. 后端创建下载任务，前端持续轮询进度。
8. 下载完成后，用户点击保存到本地。

AI 总结流程：

1. 用户粘贴公开视频链接并完成解析。
2. 用户点击“AI 总结”。
3. 后端校验 URL、免费额度和视频时长。
4. 后端通过 `yt-dlp` 获取平台原生字幕。
5. 后端清洗、去重并聚合字幕段落。
6. 后端调用 DeepSeek V4-Flash 生成总结和思维导图。
7. 前端展示要点、字幕、思维导图和对话 Tab。
8. 用户可以继续基于字幕向 AI 追问。

## 5. UI 方向

参考网站 `https://ai.codefather.cn/painting` 的整体结构：顶部导航、首屏大标题、核心输入框、卡片式内容区、清爽白底和蓝色科技感。

本项目在此基础上做付费转化增强：

- 主色从纯蓝升级为蓝紫渐变。
- Pro、推荐、定价 CTA 使用金色强调。
- 页面保留简洁白底、圆角卡片、柔和阴影和清晰留白。
- 首屏强调“一键保存高清原片”和“支持 1000+ 平台”。
- 定价区展示 Free、Pro 月付、Pro 年付，为后续商业化预留。

## 6. 风险与限制

- `yt-dlp` 支持平台很多，但平台规则变化频繁，需要定期升级。
- YouTube、B站高画质、抖音、Instagram、X 等场景可能需要 cookies 或登录态。
- 大文件会消耗磁盘空间，需要限制并发、时长和保留时间。
- AI 总结 MVP 依赖平台原生字幕，无字幕视频暂不支持。
- AI 总结会把字幕内容发送给 DeepSeek 生成结果，需要提示用户只处理有权查看和分析的内容。
- 当前免费额度按内存中的 IP 计数，服务重启会清空，不适合多进程生产部署。
- 本站不应绕过 DRM，不应鼓励用户下载无权保存或传播的内容。
- 生产环境需要加入更强的限流、日志脱敏、投诉机制和合规提示。

## 7. 后续扩展方向

- 用户上传 cookies，支持更多高画质和登录态资源。
- 批量下载队列。
- ASR 兜底，让无字幕视频也能总结。
- 字幕下载、字幕翻译和双语字幕压制。
- 总结结果导出 Markdown / PDF / Word。
- 账号系统、套餐权益、支付能力。
- Redis/Celery 任务队列，支持多 worker 和横向扩展。
